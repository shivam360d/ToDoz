Todoz
=======

## Overview

It is a **REST API** which is based on the principle of **CRUD**.
This api enables a user to :
  * create 
  * update
  * delete 
todos.
Each todo consists of three elements.First is the todo which a user wants to store.Second is the completed property which is set to true or false based on the status of the task.And third is the completedAt property which gets updated if a user completes a job.

---

User can get registered by signing in by providing their email. Moreover, it employs **authentication** via tokens using **json-web-tokens**.
